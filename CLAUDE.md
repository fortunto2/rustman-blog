# CLAUDE.md — rustman-blog

Personal build-in-public blog at **rustman.org**. Astro 5 static site deployed to Cloudflare Pages.

## Stack

- **Framework:** Astro 5 (static output)
- **Styling:** Tailwind CSS 4 + @tailwindcss/typography
- **Markdown:** markdown-it + highlight.js (github/github-dark themes) + [[wikilinks]]
- **Search:** Pagefind (client-side, indexed at build)
- **Deploy:** Cloudflare Pages (free tier)
- **Domain:** rustman.org

## Content Pipeline

Content lives in `../wiki/` (solopreneur KB). Blog pulls only `publish: true` pages.

```
../wiki/*.md         → collect-content.ts → src/content/{wiki,projects,posts}/
../1-methodology/stacks/*.yaml → collect-stacks.ts → src/content/stacks/
                                                    → Astro build → dist/
```

**Stacks pipeline:** collect-stacks.ts parses YAML frontmatter (name, platform, language, framework), generates tags, wraps as markdown with yaml code block. `libraries.yaml` is special — rendered as tables by category instead of raw YAML.

## Routes

```
/                    → homepage: avatar + project cards + recent feed
/wiki/               → knowledge base: Topics, Concepts, Summaries + Pagefind search
/wiki/{slug}         → wiki page
/projects/           → build-in-public projects
/projects/{slug}     → project page
/stacks/             → stack templates grid + libraries section
/stacks/{slug}       → stack page (yaml code block with copy button)
/stacks/libraries    → curated libraries as tables by category
/about               → full manifesto, background table, social links, avatar
/rss.xml             → RSS feed (wiki + projects + posts)
/llms.txt            → AI content map (wiki + projects + stacks)
/sitemap-index.xml   → auto-generated
/robots.txt          → crawl rules
/404                 → custom 404
```

## Commands

```bash
# From solopreneur root (preferred):
make blog-publish    # collect + build + commit + push + deploy (ONE COMMAND)
make blog-build      # collect + build (local only)
make blog-dev        # collect + dev server :4321

# From blog/ dir:
pnpm collect         # pull publish:true from ../wiki/ + stacks
pnpm build           # Astro build + Pagefind
pnpm deploy          # build + wrangler pages deploy
pnpm links           # check broken wikilinks
```

## Frontmatter for publishing

Wiki pages need `publish: true` in frontmatter to appear on blog:

```yaml
---
type: concept
title: "Page title"
description: "One sentence for SEO and RSS"
created: 2026-04-07
tags: [tag1, tag2]
publish: true              # required
publish_as: project        # optional: routes to /projects/ instead of /wiki/
source_url: "https://..."  # optional: attribution link
source_path: "1-method/..."# optional: path to raw source
---
```

Content routing:
- `publish_as: project` → `/projects/{slug}`
- `publish_as: post` → `/posts/{slug}` (homepage feed)
- Everything else → `/wiki/{slug}` (knowledge base)

## Social links

Configured in `config.ts`: GitHub, LinkedIn, Telegram, YouTube, Instagram. Footer shows GitHub, LinkedIn, Telegram, RSS, llms.txt.

## Don't

- Don't edit `src/content/` manually — generated by `pnpm collect`
- Don't commit private content (CRM, visa, personal tagged pages)
- Don't add SSR — static is simpler, free, faster
- Don't forget `description:` in wiki frontmatter — needed for RSS and SEO
- Don't add Twitter/X link — no active account
